🎮 Tutorial: Criando o Jogo Termo com React

Aprenda a construir um jogo interativo usando React, com ênfase em React Fragments

1. Introdução

Bem-vindo a este tutorial, onde embarcaremos na jornada de recriar o popular jogo de adivinhação de palavras, o Termo, utilizando a biblioteca React. Nosso objetivo é construir uma aplicação web funcional e didática, inspirada na simplicidade e clareza do tutorial oficial de Tic-Tac-Toe do React.

Ao longo deste guia, faremos uso dos arquivos de projeto já fornecidos (TermoGame.html, TermoGame.css, TermoGame.jsx) e daremos uma atenção especial à implementação de React Fragments, uma funcionalidade essencial para garantir que os componentes React retornem um único elemento raiz sem adicionar nós desnecessários ao DOM.

2. Configuração Inicial

Para iniciar nosso projeto, é fundamental compreender como os arquivos fornecidos se interligam para formar a base da aplicação. A estrutura é intencionalmente simples, permitindo que o foco permaneça na lógica do React.

3. Componentes React

Nossa aplicação é modularizada em componentes React, cada um com uma responsabilidade específica. Essa abordagem facilita a manutenção e a compreensão do código.

📦 O que é um Componente React?

Um componente React é uma função JavaScript que retorna JSX (uma sintaxe que se assemelha a HTML). Componentes permitem que você divida a interface em partes reutilizáveis e gerenciáveis.

3.1. Componente Square (Quadrado da Letra)

O componente Square é o elemento mais básico do nosso tabuleiro. Ele é responsável por exibir uma única letra e aplicar uma cor de fundo que reflete o status da letra (correta, no lugar errado ou incorreta). Ele recebe as propriedades letter e color.

function Square({ letter, color }) {
  return <div className={`square ${color || ''}`}>{letter}</div>;
}

3.2. Componente Row (Linha do Tabuleiro)

O componente Row representa uma linha completa do tabuleiro do jogo Termo, composta por cinco Squares. Ele recebe as propriedades guess (a palavra tentada) e colors (um array com as cores correspondentes a cada letra).

⭐ React Fragments em Ação: Aqui, utilizamos um React Fragment (<>...</>) para agrupar os Squares. Isso é crucial porque um componente React deve retornar um único elemento raiz. O Fragment nos permite fazer isso sem introduzir um div extra no DOM, o que poderia afetar a estilização ou a estrutura semântica. 
function Row({ guess, colors }) {
  return (
    <>
      {[...Array(5)].map((_, i) => (
        <Square 
          key={i} 
          letter={guess ? guess[i] : ''} 
          color={colors ? colors[i] : ''} 
        />
      ))}
    </>
  );
}

3.3. Componente Keyboard (Teclado Virtual)

O Keyboard é o componente que permite ao jogador interagir com o jogo, digitando letras e acionando as funções de ENTER e BACKSPACE. Ele recebe onKeyClick (uma função para lidar com o clique nas teclas) e colorsByKey (um mapa de cores para cada tecla).

⭐ React Fragments em Ação: Assim como o Row, o Keyboard também utiliza React Fragments (<>...</>) para agrupar as diferentes linhas do teclado. Cada linha é um div que contém os botões das teclas. O uso do Fragment aqui garante que o componente Keyboard retorne uma única estrutura semântica, mesmo que visualmente ele seja composto por múltiplos divs de linhas. 
function Keyboard({ onKeyClick, colorsByKey }) {
  const linhas = [
    ['Q', 'W', 'E', 'R', 'T', 'Y', 'U', 'I', 'O', 'P'],
    ['A', 'S', 'D', 'F', 'G', 'H', 'J', 'K', 'L'],
    ['ENTER', 'Z', 'X', 'C', 'V', 'B', 'N', 'M', 'BACKSPACE']
  ];

  return (
    <>
      {linhas.map((linha, i) => (
        <div key={i} className="board-row">
          {linha.map((tecla) => (
            <button 
              key={tecla} 
              className={`key ${colorsByKey[tecla] || ''}`} 
              onClick={() => onKeyClick(tecla)}
            >
              {tecla === 'BACKSPACE' ? '⌫' : tecla}
            </button>
          ))}
        </div>
      ))}
    </>
  );
}

3.4. Componente Game (Componente Principal)

O componente Game é o orquestrador de toda a aplicação. Ele gerencia o estado do jogo, incluindo as tentativas (attempts), a linha e coluna atuais (currentRow, currentCol), o status de fim de jogo (gameOver) e o mapa de cores das teclas (colorMap).

⭐ React Fragments como Elemento Raiz: É no componente Game que o uso de React Fragments se torna mais evidente como elemento raiz. Conforme a regra do React, um componente deve retornar um único elemento. No nosso caso, o Game precisa renderizar o título, a mensagem de status, o tabuleiro e o teclado. Em vez de envolver tudo em um div extra, que não teria propósito semântico, utilizamos o React Fragment (<>...</>) como o elemento raiz do retorno do componente. Isso agrupa todos os elementos visuais de forma eficiente e limpa.

4. Renderização do Aplicativo

Finalmente, para que nossa aplicação React seja exibida no navegador, precisamos renderizá-la no DOM. Isso é feito no final do arquivo TermoGame.jsx.

Utilizamos createRoot do react-dom/client para criar uma raiz de renderização no elemento com id="root" no nosso HTML. Em seguida, root.render é chamado para renderizar o componente Game dentro de React.StrictMode. O StrictMode é uma ferramenta útil para identificar problemas potenciais na aplicação durante o desenvolvimento.

const root = createRoot(document.getElementById("root"));
root.render(
  <React.StrictMode>
    <Game />
  </React.StrictMode>
);
